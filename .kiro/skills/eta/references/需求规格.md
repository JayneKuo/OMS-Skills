# ETA Skill 需求规格

## 一、定位

eta 是 OMS Agent 的时效计算能力，基于 8 组件 ETA 公式计算预估送达时间。

## 二、ETA 公式

```
ETA_total = T_queue + T_cutoff_wait + T_process + T_handover + T_transit + T_last_mile + T_weather + T_risk_buffer
```

### 8 组件说明

| 编号 | 组件 | 中文名 | 说明 |
|------|------|--------|------|
| 1 | T_queue | 排队等待时间 | 与仓当前积压量相关 |
| 2 | T_cutoff_wait | 截单等待时间 | 超过截单时间需等待下一作业窗口 |
| 3 | T_process | 仓内处理时间 | 拣货、复核、打包、称重、贴面单 |
| 4 | T_handover | 交接时间 | 打包完成到承运商揽收 |
| 5 | T_transit | 干线运输时间 | 揽收到目的地转运中心 |
| 6 | T_last_mile | 末端配送时间 | 转运中心到客户签收 |
| 7 | T_weather | 天气影响时间 | 极端天气导致的额外延迟 |
| 8 | T_risk_buffer | 风险缓冲时间 | 基于历史数据的风险缓冲 |

## 三、风险化口径

| 口径 | 百分位 | 乘数系数 | 用途 |
|------|--------|----------|------|
| 乐观 | P50 | 1.0 | 前端展示 |
| 标准 | P75 | 1.15 | 分仓决策 |
| 保守 | P90 | 1.35 | SLA 承诺 |

## 四、风险修正

```
ETA_adjusted = ETA_base × (1 + risk_factor)
risk_factor = max(f_weather, f_congestion, f_carrier_risk)
```

| 风险因子 | 触发条件 | 默认值 |
|----------|----------|--------|
| f_weather | 天气预警 | 0.3（暴雨）、0.5（暴雪/台风） |
| f_congestion | 大促/拥堵 | 0.2（普通）、0.5（高峰） |
| f_carrier_risk | 准点率 < 80% | (1 − 准点率) × 0.5 |

## 五、OnTimeProbability

```
OnTimeProbability = P(ETA_actual ≤ SLA_deadline)
```

阈值：0.85，低于此值标记"时效风险"。

## 六、样本不足回退

| 步骤 | 条件 | 动作 |
|------|------|------|
| 1 | 分桶样本 ≥ 30 | 使用统计值 |
| 2 | 样本不足 | 向上回退分桶 |
| 3 | 所有分桶不足 | 承运商 API × 1.2 |
| 4 | API 不可用 | 系统默认（同州1天/邻州2天/跨区3天/远距5天） |

## 七、美国市场默认 Transit Time

基于州级距离分段：
- 同州：Ground 1天 / Express 0.5天 / Priority 次日
- 邻州：Ground 2天 / Express 1天 / Priority 次日
- 跨区：Ground 3天 / Express 1.5天 / Priority 1天
- 远距：Ground 5天 / Express 2天 / Priority 1.5天

## 八、典型案例

上海仓→成都，顺丰，工作日 14:00 下单，截单 16:00：
- T_queue=0.5h, T_cutoff_wait=0h, T_process=2h, T_handover=1h
- T_transit=36h, T_last_mile=4h, T_weather=0h, T_risk_buffer=2h
- ETA_total = 45.5h ≈ 1.9天
- OnTimeProbability(SLA 3天) = 0.94
