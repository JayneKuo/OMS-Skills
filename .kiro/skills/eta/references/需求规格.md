# ETA Skill 需求规格

## 一、定位

eta 是 OMS Agent 的时效计算能力，基于 8 组件 ETA 公式计算预估送达时间，输出 P50/P75/P90 三口径结果和准时概率。

---

## 二、ETA 公式

```
ETA_total = T_queue + T_cutoff_wait + T_process + T_handover
          + T_transit + T_last_mile + T_weather + T_risk_buffer
```

### 8 组件说明

| 编号 | 组件 | 中文名 | 计算逻辑 | 降级条件 |
|------|------|--------|----------|----------|
| 1 | T_queue | 排队等待 | backlog_orders / processing_speed | processing_speed 未知时默认 0.5h |
| 2 | T_cutoff_wait | 截单等待 | 超截单时间则等到下一作业窗口开始 | 无降级 |
| 3 | T_process | 仓内处理 | 仓库配置的标准处理时间 | 默认 2h |
| 4 | T_handover | 交接 | 距下一次承运商揽收的小时数 | 默认 1h |
| 5 | T_transit | 干线运输 | 承运商 API × 1.2 校准；无 API 则查默认表 | 无 API 时用默认表，标记降级 |
| 6 | T_last_mile | 末端配送 | 按服务级别查默认表 | 结构性估算，不影响置信度 |
| 7 | T_weather | 天气影响 | T_transit × f_weather | f_weather=0 时为 0 |
| 8 | T_risk_buffer | 风险缓冲 | T_transit × (1 − on_time_rate) × 0.5 | on_time_rate ≥ 1 时为 0 |

---

## 三、风险化口径（动态乘数）

P75/P90 乘数随承运商准点率动态调整，准点率越低分位数越分散：

```
spread = (1 − on_time_rate) × 2
P50 乘数 = 1.00
P75 乘数 = 1.15 + spread × 0.3
P90 乘数 = 1.35 + spread × 0.6
```

| 准点率 | P50 | P75 | P90 | 用途 |
|--------|-----|-----|-----|------|
| 0.95 | 1.00 | 1.18 | 1.41 | 前端展示 / 分仓决策 / SLA 承诺 |
| 0.92（基准） | 1.00 | 1.20 | 1.45 | — |
| 0.85 | 1.00 | 1.24 | 1.53 | — |
| 0.70 | 1.00 | 1.33 | 1.71 | — |

> 旧版固定乘数（1.00/1.15/1.35）已废弃。

---

## 四、风险修正

```
ETA_adjusted = ETA_base × (1 + risk_factor)
risk_factor = max(f_weather, f_congestion, f_carrier_risk)
```

| 风险因子 | 触发条件 | 系数 |
|----------|----------|------|
| f_weather | 暴雨 | 0.3 |
| f_weather | 暴雪 / 台风 | 0.5 |
| f_congestion | 普通大促 | 0.2 |
| f_congestion | 高峰（黑五/双11） | 0.5 |
| f_carrier_risk | 准点率 < 80% | (1 − 准点率) × 0.5 |

## 五、节假日/旺季自动检测

当请求传入 `order_month` + `order_day` 且未手动设置 `congestion_level` 时，自动推断：

| 时间段 | 级别 |
|--------|------|
| 11/25–11/30（黑五/网一） | peak |
| 12/15–12/31（圣诞旺季） | peak |
| 1/1–1/3（元旦） | peak |
| 2/10–2/16（情人节） | normal_promo |
| 5/20–5/27（阵亡将士纪念日） | normal_promo |
| 7/1–7/7（独立日） | normal_promo |
| 9/1–9/7（劳工节） | normal_promo |
| 10/28–10/31（万圣节） | normal_promo |

手动传入 `congestion_level` 时，自动检测不覆盖。

## 六、OnTimeProbability

```
OnTimeProbability = P(ETA_actual ≤ SLA_deadline)
```

基于 P50/P90 线性插值估算：
- SLA ≥ P90 → 0.91–0.99
- P50 ≤ SLA < P90 → 线性插值 0.50–0.90
- SLA < P50 → 线性插值 0.05–0.50

阈值：< 0.85 标记 `on_time_risk=true`。

---

## 七、置信度分级

| 级别 | 条件 |
|------|------|
| high | 承运商 API 提供 transit time，仓库数据完整 |
| medium | 使用默认 transit 表，其他字段正常 |
| estimated | t_queue 也降级（仓库无处理速度数据）或多字段缺失 |

> `t_last_mile` 为结构性估算，不计入置信度判断。

---

## 八、数据降级策略

| 步骤 | 条件 | 动作 |
|------|------|------|
| 1 | 承运商 API 返回 transit_hours | 使用 API 值 × 1.2 校准，confidence=high |
| 2 | API 不可用 | 查默认 transit 表，confidence=medium |
| 3 | 仓库 processing_speed 未知 | t_queue 默认 0.5h，confidence=estimated |

---

## 九、美国市场默认 Transit Time

| 距离分段 | 判断规则 | Ground | Express | Priority |
|----------|----------|--------|---------|----------|
| same_state | 同州 | 24h | 12h | 8h |
| adjacent | 同区域不同州 | 48h | 24h | 12h |
| cross | 邻接区域 | 72h | 36h | 24h |
| remote | 其他 | 120h | 48h | 36h |

区域划分：WEST / CENTRAL / SOUTH / EAST（详见 engine.py `US_REGIONS`）

## 十、典型案例

**NJ → CA，UPS Ground，6月平日 14:00 下单，承运商准点率 0.92：**
- T_queue=0h（无积压）, T_cutoff_wait=0h, T_process=2h, T_handover=1h
- T_transit=72h（默认 cross 段）, T_last_mile=6h
- ETA_base ≈ 81h，risk_factor=0.04（准点率修正）
- ETA_adjusted ≈ 84h
- P50=84h / P75=101h / P90=122h
- confidence=medium

**同路线，黑五（11/27）下单：**
- f_congestion=0.5（peak 自动检测）
- ETA_adjusted ≈ 126h
- P75=257h / P90=318h
