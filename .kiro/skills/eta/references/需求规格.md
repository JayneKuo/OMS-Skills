# eta 需求规格

## 1. 模块定位

`eta` 是 OMS Agent 的时效计算能力，基于 8 组件 ETA 公式计算预估送达时间，输出 P50/P75/P90 三口径结果、准时概率和置信度，为分仓推荐、发货方案和 SLA 承诺提供时效依据。

### 1.1 核心目标

当上游提供发货州、收货州、承运商、服务级别、仓库上下文时，eta 应能回答：

- 这单预计几天到
- P50/P75/P90 各口径分别是多少
- 在给定 SLA 下准时概率是多少
- 当前计算结果的置信度如何
- 是否存在时效风险

### 1.2 模块边界

**负责：**
- 8 组件 ETA 计算
- P50/P75/P90 动态三口径
- 风险修正（天气/拥堵/承运商准点率）
- 节假日/旺季自动检测
- OnTimeProbability 计算
- 置信度分级
- 数据降级与透明标注

**不负责：**
- 运费计算（→ shipping_rate）
- 综合成本评分（→ cost）
- 仓库推荐（→ warehouse_allocation）
- 实际履约执行
- 历史数据统计（当前为 MVP，依赖默认表）

## 2. 核心能力清单

| 编号 | 能力 | 说明 |
|------|------|------|
| ETA-1 | 8 组件 ETA 计算 | T_queue + T_cutoff_wait + T_process + T_handover + T_transit + T_last_mile + T_weather + T_risk_buffer |
| ETA-2 | P50/P75/P90 动态三口径 | 乘数随承运商准点率动态调整，准点率越低分位数越分散 |
| ETA-3 | 风险修正 | 天气/拥堵/承运商准点率三因子取最大值修正基础 ETA |
| ETA-4 | 节假日/旺季自动检测 | 传入 order_month + order_day 时自动推断 congestion_level |
| ETA-5 | OnTimeProbability 计算 | 基于 P50/P90 线性插值估算准时概率 |
| ETA-6 | 置信度分级 | high（承运商 API）/ medium（默认表）/ estimated（多字段缺失） |
| ETA-7 | 数据降级策略 | 4 步降级，每步透明标注数据来源 |

## 5. ETA 公式与 8 组件

```
ETA_total = T_queue + T_cutoff_wait + T_process + T_handover
          + T_transit + T_last_mile + T_weather + T_risk_buffer
```

| 组件 | 计算逻辑 | 降级条件 | 降级默认值 |
|------|----------|----------|-----------|
| T_queue | backlog_orders / processing_speed | processing_speed = 0 | 0.5h，标记降级 |
| T_cutoff_wait | order_hour < cutoff_hour → 0；否则 → 24 - order_hour + work_start_hour | 无 | — |
| T_process | warehouse.process_time_hours | 无 | 默认 2h |
| T_handover | carrier_ctx.next_pickup_hours | 无 | 默认 1h |
| T_transit | 承运商 API × 1.2；无 API 则查默认 transit 表 | 无 API 时用默认表 | 见第 12 节 |
| T_last_mile | 按服务级别查默认表 | 结构性估算，不影响置信度 | Ground 6h / Express 4h / Priority 2h |
| T_weather | T_transit × f_weather | f_weather = 0 时为 0 | — |
| T_risk_buffer | T_transit × (1 − on_time_rate) × 0.5 | on_time_rate ≥ 1 时为 0 | — |

---

## 6. 动态 P50/P75/P90 乘数

P75/P90 乘数随承运商准点率动态调整，准点率越低分位数越分散：

```
spread = (1 − on_time_rate) × 2
P50 乘数 = 1.00
P75 乘数 = 1.15 + spread × 0.3
P90 乘数 = 1.35 + spread × 0.6
```

| 准点率 | P50 | P75 | P90 |
|--------|-----|-----|-----|
| 0.95 | 1.00 | 1.18 | 1.41 |
| 0.92（基准） | 1.00 | 1.202 | 1.446 |
| 0.85 | 1.00 | 1.24 | 1.53 |
| 0.70 | 1.00 | 1.33 | 1.71 |

> v1.0 固定乘数（1.00/1.15/1.35）已废弃。

## 7. 风险修正

```
ETA_adjusted = ETA_base × (1 + risk_factor)
risk_factor = max(f_weather, f_congestion, f_carrier_risk)
```

| 风险因子 | 触发条件 | 系数 |
|----------|----------|------|
| f_weather | weather_alert = rain | 0.3 |
| f_weather | weather_alert = snow / typhoon | 0.5 |
| f_congestion | congestion_level = normal_promo | 0.2 |
| f_congestion | congestion_level = peak | 0.5 |
| f_carrier_risk | on_time_rate < 0.8 | (1 − on_time_rate) × 0.5 |

## 8. 节假日/旺季自动检测

当请求传入 `order_month` + `order_day` 且 `congestion_level = "none"` 时，自动推断：

| 时间段 | 节日 | 级别 |
|--------|------|------|
| 11/25–11/30 | 黑五/网一 | peak |
| 12/15–12/31 | 圣诞旺季 | peak |
| 1/1–1/3 | 元旦 | peak |
| 2/10–2/16 | 情人节 | normal_promo |
| 5/20–5/27 | 阵亡将士纪念日 | normal_promo |
| 7/1–7/7 | 独立日 | normal_promo |
| 9/1–9/7 | 劳工节 | normal_promo |
| 10/28–10/31 | 万圣节 | normal_promo |

**触发逻辑：**
1. 仅当 `order_month` 和 `order_day` 均不为 null 时执行检测
2. 仅当 `congestion_level = "none"` 时自动覆盖（手动传入的值不被覆盖）
3. 未命中任何时间段时保持 "none"

## 9. 置信度分级

| 级别 | 判断条件 |
|------|----------|
| high | 承运商 API 提供 transit time，且 processing_speed > 0 |
| medium | 使用默认 transit 表（t_transit 降级），其他字段正常 |
| estimated | t_queue 也降级（processing_speed = 0）或其他多字段缺失 |

**特别说明：**
- `t_last_mile` 为结构性估算（始终使用默认表），不计入置信度判断
- 置信度以 `meaningful_degraded = degraded_fields - {t_last_mile}` 为判断依据

---

## 10. 数据降级策略

| 步骤 | 条件 | 动作 | confidence |
|------|------|------|-----------|
| 1 | carrier_ctx.api_transit_hours 有值且 > 0 | 使用 API 值 × 1.2 校准 | high |
| 2 | API 不可用 | 查默认 transit 表（按州级距离分段） | medium |
| 3 | warehouse.processing_speed = 0 | t_queue 默认 0.5h | estimated |
| 4 | 多字段缺失 | 全部使用默认值 | estimated |

## 11. OnTimeProbability 计算

```
OnTimeProbability = P(ETA_actual ≤ SLA_deadline)
```

基于 P50/P90 线性插值估算：

| SLA 范围 | 计算方式 |
|----------|----------|
| SLA ≥ P90 | 0.91 + min((SLA-P90)/P90 × 0.05, 0.04)，上限 0.99 |
| P50 ≤ SLA < P90 | 0.50 + (SLA-P50)/(P90-P50) × 0.40 |
| SLA < P50 | max(SLA/P50 × 0.50, 0.05) |

**阈值：** on_time_probability < 0.85 → on_time_risk = true

## 12. 美国市场默认 Transit Time

### 距离分段规则

| 分段 | 判断条件 |
|------|----------|
| same_state | 发货州 = 收货州 |
| adjacent | 同区域不同州 |
| cross | 邻接区域（WEST↔CENTRAL、CENTRAL↔SOUTH/EAST、SOUTH↔EAST） |
| remote | 其他 |

区域划分：
- WEST：CA OR WA NV AZ UT CO NM ID MT WY HI AK
- CENTRAL：TX OK KS NE SD ND MN IA MO AR LA WI IL IN MI OH
- SOUTH：FL GA SC NC VA WV KY TN AL MS MD DE DC
- EAST：NY NJ PA CT MA RI VT NH ME

### 默认时间表（小时）

| 分段 | Ground | Express | Priority |
|------|--------|---------|----------|
| same_state | 24 | 12 | 8 |
| adjacent | 48 | 24 | 12 |
| cross | 72 | 36 | 24 |
| remote | 120 | 48 | 36 |

## 16. 典型案例

### 案例 1：NJ → CA，UPS Ground，6 月平日，准点率 0.92

输入：origin=NJ, dest=CA, service=Ground, order_month=6, order_day=15, order_hour=14, on_time_rate=0.92

| 组件 | 值 |
|------|-----|
| T_queue | 0h（无积压） |
| T_cutoff_wait | 0h（14 < 16） |
| T_process | 2h |
| T_handover | 1h |
| T_transit | 120h（cross 段默认） |
| T_last_mile | 6h |
| T_weather | 0h |
| T_risk_buffer | 9h（120 × 0.08 × 0.5 × 1.5） |

- ETA_base ≈ 138h，risk_factor ≈ 0.04（f_carrier_risk）
- ETA_adjusted ≈ 143h
- P50 ≈ 134h / P75 ≈ 161h / P90 ≈ 194h
- confidence = medium（默认 transit 表）

### 案例 2：NJ → CA，UPS Ground，黑五（11/27），准点率 0.85

输入：origin=NJ, dest=CA, service=Ground, order_month=11, order_day=27, on_time_rate=0.85

- congestion_level 自动检测为 peak（f_congestion=0.5）
- risk_factor = 0.5（peak 拥堵为最大因子）
- ETA_adjusted ≈ 207h（基础 ETA × 1.5）
- P50 ≈ 207h / P75 ≈ 257h / P90 ≈ 317h
- confidence = medium
