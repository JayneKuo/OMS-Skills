# cost 需求规格

## 1. 模块定位

`cost` 是 OMS Agent 的综合成本计算能力，将运费、仓操费、调拨费、拆单惩罚、容量惩罚、风险成本统一为可排序的综合评分，支持多方案对比和推荐排序。

### 1.1 核心目标

当上游提供多个发货方案（不同仓库/承运商/服务组合）时，cost 应能回答：

- 哪个方案综合成本最优
- 各方案的成本构成明细是什么
- 在不同业务场景（时效优先/成本优先）下推荐结果如何变化
- 当前评分结果的置信度如何

### 1.2 模块边界

**负责：**
- 6 项成本汇总计算
- 4 维加权评分
- 场景预设权重
- Min-Max / 参考值归一化
- 容量惩罚计算
- 拆单惩罚计算
- 多方案排序与推荐

**不负责：**
- 运费计算（→ shipping_rate）
- 时效计算（→ eta）
- 仓库推荐（→ warehouse_allocation）
- 装箱计算（→ cartonization）

---

## 2. 核心能力清单

| 编号 | 能力 | 说明 |
|------|------|------|
| COST-1 | 6 项成本汇总 | Freight + Warehouse + Transfer + Penalty_split + Penalty_capacity + Risk |
| COST-2 | 4 维加权评分 | 成本/时效/准时率/容量，支持场景预设和自定义权重 |
| COST-3 | 场景预设 | balanced / cost_sensitive / time_sensitive / reliability 四种开箱即用配置 |
| COST-4 | 权重自动归一化 | 权重总和不等于 1.0 时自动归一化，不报错 |
| COST-5 | Min-Max 归一化 | 方案集内相对归一化，越低越好的维度反向映射 |
| COST-6 | 参考值归一化 | 指定 cost_ref_max / eta_ref_max 时使用绝对归一化 |
| COST-7 | 容量惩罚 | 4 梯度惩罚函数，利用率 > 70% 开始惩罚 |
| COST-8 | 拆单惩罚 | (N_warehouses - 1) × split_penalty_unit |
| COST-9 | 多方案排序 | Score 降序排列，rank=1 为推荐方案 |
| COST-10 | 单方案降级标注 | 仅 1 个方案时归一化无意义，标记 degraded 并提示 |

## 3. 输入规格

### 3.1 CostRequest

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| plans | list[PlanInput] | ✅ | — | 待评分方案列表，至少 1 个 |
| weights | ScoreWeights | — | balanced 默认值 | 自定义权重；preset 指定时被覆盖 |
| preset | str\|None | — | null | 场景预设：balanced/cost_sensitive/time_sensitive/reliability |
| split_penalty_unit | Decimal | — | 5 | 拆单惩罚单价（元/仓） |
| sla_hours | Decimal | — | 72 | SLA 时限（小时） |
| cost_ref_max | Decimal\|None | — | null | 成本归一化参考最大值；指定时使用参考值归一化 |
| eta_ref_max | Decimal\|None | — | null | 时效归一化参考最大值；指定时使用参考值归一化 |

### 3.2 PlanInput

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| plan_id | str | — | 方案唯一标识 |
| plan_name | str | "" | 方案名称（如"UPS Ground / Ontario 仓"） |
| freight_order | Decimal | 0 | 订单运费（所有包裹运费之和） |
| cost_warehouse | Decimal | 0 | 仓操费（拣货/打包/称重等） |
| cost_transfer | Decimal | 0 | 调拨费（调拨合发场景） |
| n_warehouses | int | 1 | 发货仓数量，> 1 时触发拆单惩罚 |
| capacity_utilization | Decimal | 0 | 仓容量利用率（0~1），> 0.70 时触发容量惩罚 |
| cost_risk | Decimal | 0 | 风险成本（如时效风险补偿） |
| eta_hours | Decimal | 0 | 预估送达时间（小时），来自 eta skill |
| on_time_probability | Decimal | 0 | 准时送达概率（0~1），来自 eta skill |
| remain_capacity_pct | Decimal\|None | null | 剩余容量百分比（0~1）；null 时自动计算为 1 - capacity_utilization |

### 3.3 ScoreWeights

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| w_cost | Decimal | 0.40 | 成本维度权重 |
| w_eta | Decimal | 0.30 | 时效维度权重 |
| w_ontime | Decimal | 0.15 | 准时率维度权重 |
| w_cap | Decimal | 0.15 | 容量维度权重 |

---

## 4. 场景预设

| 预设名 | w_cost | w_eta | w_ontime | w_cap | 适用场景 |
|--------|--------|-------|----------|-------|---------|
| balanced | 0.40 | 0.30 | 0.15 | 0.15 | 默认均衡，无特殊偏好 |
| cost_sensitive | 0.60 | 0.20 | 0.10 | 0.10 | 成本压力大，时效要求宽松 |
| time_sensitive | 0.20 | 0.50 | 0.20 | 0.10 | 时效优先，如大促/急单 |
| reliability | 0.25 | 0.25 | 0.40 | 0.10 | 可靠性优先，如高价值商品 |

**优先级：** preset > weights（指定 preset 时忽略 weights 字段）

---

## 6. Cost_total 公式

```
Cost_total = Freight_order + Cost_warehouse + Cost_transfer
           + Penalty_split + Penalty_capacity + Cost_risk
```

---

## 7. Score 公式

```
Score = w_cost × Normalize(Cost_total)
      + w_eta  × Normalize(ETA)
      + w_ontime × OnTimeProbability
      + w_cap  × RemainCapacity
```

---

## 8. 归一化规则

### Min-Max 反向归一化（默认）

```
Normalize(x) = 1 - (x - min) / (max - min)
```

- 越低越好的维度（成本、时效）反向映射为越高分
- max = min 时（单方案或所有方案相同）返回 1.0，标记 normalization 降级

### 参考值归一化（指定 cost_ref_max / eta_ref_max 时）

```
Normalize(x) = 1 - x / ref_max
```

- 适用于跨批次方案对比，保持评分可比性
- 结果下限为 0

---

## 9. 容量惩罚函数

| 梯度 | 利用率区间 | 惩罚公式 |
|------|-----------|----------|
| Tier 1 | 0% ~ 70% | 0 |
| Tier 2 | 70% ~ 85% | (u - 0.70) × 10 |
| Tier 3 | 85% ~ 95% | 1.5 + (u - 0.85) × 30 |
| Tier 4 | 95% ~ 100% | 4.5 + (u - 0.95) × 100 |

---

## 10. 权重校验与归一化

- 四个权重总和必须 > 0
- 总和不等于 1.0 时自动归一化（不报错）：每个权重除以总和，最后一项用 1 减去前三项之和确保精确
- 总和 = 0 时返回错误

## 11. 与上下游 Skill 的关系

| 方向 | Skill | 交互内容 |
|------|-------|---------|
| 上游 | eta | 提供 eta_hours 和 on_time_probability |
| 上游 | shipping_rate | 提供 freight_order |
| 上游 | warehouse_allocation | 提供候选方案列表（仓库/承运商组合） |
| 下游 | shipping_plan workflow | 使用 cost 评分生成 Top-3 发货方案推荐 |
| 下游 | oms-agent | 直接展示综合评分结果 |

## 12. 适用场景

### 直接触发

| 用户问法 | 触发 |
|---------|------|
| "哪个方案综合最优" / "帮我比较这几个发货方案" | ✅ |
| "时效优先的话选哪个" / "成本最低的方案是哪个" | ✅（指定 preset） |
| "这几个方案的成本明细是什么" | ✅ |

### 不触发（转其他 Skill）

| 用户问法 | 转给 |
|---------|------|
| "运费多少" | shipping_rate |
| "预计几天到" | eta |
| "应该分哪个仓" | warehouse_allocation |

---

## 13. 验收标准

| 编号 | 验收项 |
|------|--------|
| AC-1 | time_sensitive 预设下，ETA 最短的方案排名第一 |
| AC-2 | cost_sensitive 预设下，Cost_total 最低的方案排名第一 |
| AC-3 | 权重 4/3/1.5/1.5 自动归一化为 0.40/0.30/0.15/0.15 |
| AC-4 | 单方案时 degraded_fields 包含 normalization，explanation 包含提示 |
| AC-5 | 容量利用率 0.90 时 Penalty_capacity = 1.5 + (0.90-0.85) × 30 = 3.0 |
| AC-6 | n_warehouses=3, split_penalty_unit=5 时 Penalty_split = 10 |
| AC-7 | on_time_probability < 0.85 时 on_time_risk = true |
| AC-8 | 未指定 preset 时使用 weights 字段 |
| AC-9 | 指定 preset 时忽略 weights 字段 |
| AC-10 | plans 为空时返回 success=false 和错误信息 |

## 14. 典型案例

### 案例 1：三方案 balanced 对比

| 方案 | 运费 | ETA | 准时率 | Score | 排名 |
|------|------|-----|--------|-------|------|
| UPS Ground | $50 | 72h | 0.92 | 0.71 | 1 |
| USPS Priority | $80 | 48h | 0.88 | 0.71 | 2 |
| FedEx Express | $120 | 36h | 0.97 | 0.60 | 3 |

### 案例 2：同三方案 time_sensitive 对比

| 方案 | Score | 排名 |
|------|-------|------|
| FedEx Express（最快） | 0.79 | 1 |
| USPS Priority | 0.73 | 2 |
| UPS Ground（最慢） | 0.48 | 3 |

> 切换 preset 后推荐结果完全反转，体现场景预设的业务价值。
