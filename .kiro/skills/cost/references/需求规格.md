# Cost Skill 需求规格

## 一、定位

cost 是 OMS Agent 的综合成本计算能力，将所有成本项和时效指标统一为可排序的评分。

## 二、Cost_total 公式

```
Cost_total = Freight_order + Cost_warehouse + Cost_transfer + Penalty_split + Penalty_capacity + Cost_risk
```

| 成本项 | 定义 |
|--------|------|
| Freight_order | 订单运费（所有包裹运费之和） |
| Cost_warehouse | 仓操费（拣货、打包、称重等） |
| Cost_transfer | 调拨费（调拨合发场景） |
| Penalty_split | 拆单惩罚 = (N_warehouses - 1) × 5 元 |
| Penalty_capacity | 容量惩罚（按利用率 4 梯度） |
| Cost_risk | 风险成本 |

## 三、Score 公式

```
Score = w_cost × Normalize(Cost_total) + w_eta × Normalize(ETA) + w_ontime × OnTimeProbability + w_cap × Normalize(RemainCapacity)
```

### 默认权重

| 维度 | 权重 | 折算口径 | 方向 |
|------|------|----------|------|
| 成本 | w_cost = 0.40 | 人民币金额 | 越低越好 |
| 时效 | w_eta = 0.30 | 小时归一化 | 越快越好 |
| 准时率 | w_ontime = 0.15 | 概率值 [0,1] | 越高越好 |
| 容量 | w_cap = 0.15 | 剩余容量% | 越高越好 |

## 四、容量惩罚函数

| 梯度 | 利用率区间 | 惩罚公式 |
|------|-----------|----------|
| Tier 1 | 0% ~ 70% | 0 |
| Tier 2 | 70% ~ 85% | (utilization - 0.70) × 10 |
| Tier 3 | 85% ~ 95% | 1.5 + (utilization - 0.85) × 30 |
| Tier 4 | 95% ~ 100% | 4.5 + (utilization - 0.95) × 100 |

## 五、归一化

成本和时效使用 Min-Max 反向归一化（越低越好 → 越高分）：
```
Normalize_cost(x) = 1 - (x - min) / (max - min)   # 成本越低分越高
Normalize_eta(x) = 1 - (x - min) / (max - min)     # 时效越快分越高
```

准时率和容量直接使用原始值（已在 [0,1] 范围）。

## 六、典型案例

三方案对比，收货地成都：

| 方案 | Cost_total | ETA | OnTimeProb | RemainCap | Score |
|------|-----------|-----|------------|-----------|-------|
| A: 上海单仓 | 28.3 | 45.5h | 0.94 | 35% (1.0) | 0.834 |
| B: 成都+上海拆发 | 28.2 | 45.5h | 0.91 | 22% (0.56) | 0.768 |
| C: 成都调拨合发 | 23.7 | 60h | 0.82 | 22% (0.56) | 0.757 |

推荐方案 A，Score 最高。
