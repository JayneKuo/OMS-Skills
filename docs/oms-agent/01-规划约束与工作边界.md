# OMS Agent 规划约束与工作边界

> 本文件记录用户既定规划、固定边界、Agent 工作边界、后续更新 skill 的规则。
> 任何 skill 更新操作前必须先读取本文件。

---

## 1. 既定顶层 Skill 体系（不可修改）

以下 7 个 skill 为用户既定规划，名称、定位、职责边界不可变更：

| skill | 类型 | 定位 |
|-------|------|------|
| cartonization | 工具型 | 装箱建议/箱型选择/包裹拆分/计费重计算/特殊标记 |
| oms_query | 查询型 | 订单基础信息/状态/明细/履约状态/关键节点查询 |
| order_analysis | 分析型 | 订单异常/失败/卡单/规则命中分析，输出根因和建议 |
| warehouse_allocation | 推荐型 | 仓库分配建议，基于库存/仓能力/规则/目的地 |
| shipping_rate | 计算型 | 运费计算，基于发货地/收货地/包裹/承运商/服务 |
| eta | 计算型 | 时效计算，基于发货地/收货地/承运商/服务/日历 |
| cost | 计算型 | 综合成本计算，汇总运费/仓操费/包材/拆单惩罚 |

## 2. 既定 Workflow 体系（不可修改）

| workflow | 触发场景 |
|----------|---------|
| oms_query_workflow | 查询订单 |
| order_exception_diagnosis_workflow | 订单异常分析 |
| warehouse_allocation_recommendation_workflow | 分仓建议 |
| shipping_plan_recommendation_workflow | 发货方案推荐 |
| warehouse_plus_shipping_recommendation_workflow | 仓库+物流综合推荐 |

## 3. 已有 Skill 实现（不可推翻）

- `cartonization`：已有完整 Python 实现（cartonization_engine/），含 spec、skill 定义、脚本、测试。
- 其余 6 个 skill 状态为"规划中"。

## 4. Agent 工作边界

### 允许做的
- 盘点现有代码能力并建立索引
- 将现有能力映射到既定 skill
- 识别共享能力 / 底层依赖能力
- 梳理 workflow 对能力和 skill 的依赖
- 在既定 skill 框架内补充实现细节
- 新增共享能力层（不影响顶层 skill 定义）

### 不允许做的
- 修改顶层 skill 名称、定位、职责边界
- 新增或删除顶层 skill
- 推翻已有的 cartonization 实现
- 重新定义 workflow 体系
- 将共享能力提升为独立顶层 skill

## 5. 后续更新 Skill 的规则

1. 更新任何 skill 前，必须先读取本目录下所有文件
2. 必须基于 `oms-existing-capabilities-index.md` 中的现有能力来设计
3. 必须参照 `skill-capability-mapping.md` 确认能力归属
4. 新发现的能力只能归入 `shared-capabilities.md`，不能改动顶层 skill
5. workflow 依赖变更必须同步更新 `workflow-dependency-map.md`
6. 所有变更必须注明日期和变更原因

├── linker-module-oas-api/    # API 定义层（DTO/VO/Entity/Enum/Strategy接口/Struct）

│       ├── api/dto/           # 对外 DTO

│       ├── struct/            # 核心数据结构（OrderContext/DispatchDTO 等）
│       └── utils/             # 工具类
├── linker-module-oas-biz/     # 业务实现层
│   └── com.unis.linker.module.oas/
│       ├── annotation/        # @OasLog 日志注解
│       ├── config/            # 配置类（Nacos/Redis/Distance）
│       ├── controller/admin/  # 管理接口
│       ├── controller/app/    # 前端接口（Mapping CRUD / DispatchLog）
│       ├── controller/rpc/feign/ # RPC 接口（Dispatch/Mapping/Channel）
│       └── domain/            # 领域 DTO（ShippingMapping）
