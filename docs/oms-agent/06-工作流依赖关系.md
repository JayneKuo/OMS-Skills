# Workflow 依赖关系图

> 梳理 OMS Agent workflow 对现有能力和 skill 的依赖。
> 最后更新：2026-04-07

---

## 1. oms_query_workflow

```
触发：查询订单 / 订单状态 / 订单明细

步骤：
1. 识别订单号 → 无现有能力依赖（Agent 自身）
2. 调用 oms_query skill
   ├── dispatch_log_query (A5) ← /dispatch-log/{eventId}
   ├── dispatch_result_log (G3) ← DispatchLogDO
   ├── dispatch_status_tracking (D4) ← DispatchStatusEnum
   ├── order_item_data (H1) ← OrderItemDO
   └── delivery_status_update (F3) ← DeliveryGenerateStrategy
3. 输出结果 + 业务解释

现有能力覆盖度：高
缺失：订单头信息查询（OMS 中 OrderItemDO 有 orderNo 但无独立订单头表在 OAS 模块）
```

## 2. order_exception_diagnosis_workflow

```
触发：订单为什么失败 / 为什么没分仓 / 为什么没发货

步骤：
1. 调用 oms_query skill
   ├── dispatch_log_query (A5)
   ├── dispatch_status_tracking (D4)
   └── order_item_data (H1)
2. 调用 order_analysis skill
   ├── dispatch_process_log (G2) ← OrderContext.logs
   ├── dispatch_exception_msg (D5) ← exceptionMsg
   ├── oas_log_aspect (G1) ← @OasLog 日志
   ├── warehouse_inventory_check (B3) ← 库存校验逻辑
   ├── warehouse_fulfillment_check (B4) ← 全量发货校验
   └── 规则命中记录 ← DispatchLogDO.filterStrategies/dispatchStrategies
3. 如与仓分配有关 → 调用 warehouse_allocation skill
   └── sales_order_dispatch (A1)
4. 如与物流有关 → 调用 shipping_rate skill
   └── shipping_mapping_execute (E4)
5. 汇总输出：问题类型 / 根因 / 影响步骤 / 是否可重试 / 修复建议

现有能力覆盖度：高
缺失：error_logs 独立查询接口（当前异常信息嵌在 DispatchLogDO 中）
```

## 3. warehouse_allocation_recommendation_workflow

```
触发：这单该分哪个仓 / 推荐发货仓

步骤：
1. 调用 oms_query skill
   ├── order_item_data (H1)
   └── dispatch_log_query (A5)（如有历史记录）
2. 获取库存/仓能力
   ├── warehouse_inventory_check (B3)
   ├── warehouse_distance_data (B5)
   └── channel_query (H3)
3. 调用 warehouse_allocation skill
   ├── sales_order_dispatch (A1) ← 核心入口
   ├── warehouse_filter_by_zipcode (B1)
   ├── warehouse_filter_by_country (B2)
   ├── closest_warehouse_selection (B6)
   ├── no_split_strategy (C1) / minimal_split_strategy (C2)
   ├── custom_rule_dispatch (C5)
   ├── default_rule_config (C7) + custom_rule_config (C8)
   └── one_warehouse_backup (D1) / multi_warehouse_backup (D2)
4. 如需完整建议 → 继续调用：
   ├── cartonization skill ← cartonization_engine (独立 Python)
   ├── shipping_rate skill ← shipping_mapping_execute (E4)
   ├── eta skill ← warehouse_distance_data (B5) + carrier_data (E5)
   └── cost skill ← 汇总以上结果
5. 输出：推荐仓 / 候选仓 / 推荐承运商 / 推荐理由

现有能力覆盖度：warehouse_allocation 部分最高（18项直接复用）
缺失：多维评分排序（当前仅有 sort + points，PRD 规划的 α/β/γ/δ 加权评分未实现）
```

## 4. shipping_plan_recommendation_workflow

```
触发：这单怎么发最合适 / 走哪个承运商

步骤：
1. 调用 oms_query skill
   └── order_item_data (H1)
2. 获取 SKU 主数据
   ├── one_to_one_mapping (E1) ← SKU/UOM 映射
   └── sku_uom_mapping (H9)
3. 调用 cartonization skill
   └── cartonization_engine (独立 Python 实现)
4. 调用 shipping_rate skill
   ├── shipping_mapping_execute (E4) ← 核心
   ├── one_to_one_mapping (E1) ← Carrier/Service 映射
   ├── condition_mapping (E2)
   └── carrier_data (E5)
5. 调用 eta skill
   ├── warehouse_distance_data (B5)
   └── carrier_data (E5)
6. 调用 cost skill
   └── 汇总 shipping_rate + eta 结果
7. 输出：装箱方案 / 推荐承运商 / 推荐服务 / 预计运费 / 预计时效 / 推荐理由

现有能力覆盖度：中
缺失：运费阶梯价计算逻辑、ETA 计算逻辑、综合成本计算逻辑
```

## 5. warehouse_plus_shipping_recommendation_workflow

```
触发：从仓库到物流服务一起推荐

步骤：
1-3. 同 warehouse_allocation_recommendation_workflow 的步骤 1-3
4. 调用 cartonization skill
5. 调用 shipping_rate skill
6. 调用 eta skill
7. 调用 cost skill
8. 比较多个候选组合（仓库 × 承运商 × 服务）
9. 输出最终推荐

依赖关系 = warehouse_allocation_workflow ∪ shipping_plan_workflow
现有能力覆盖度：中（分仓部分高，运费/时效/成本部分低）
```

## 7. 各 Workflow 的现有能力缺口

| workflow | 现有能力覆盖 | 主要缺口 |
|----------|------------|---------|
| oms_query_workflow | 高 | 订单头独立查询接口 |
| order_exception_diagnosis_workflow | 高 | error_logs 独立查询 |
| warehouse_allocation_recommendation_workflow | 高（分仓部分） | 多维评分排序、容量惩罚 |
| shipping_plan_recommendation_workflow | 中 | 运费阶梯价计算、ETA 计算、成本汇总 |
| warehouse_plus_shipping_recommendation_workflow | 中 | 同上 + 多方案对比排序 |
