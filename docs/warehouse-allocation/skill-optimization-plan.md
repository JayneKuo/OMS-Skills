# 装箱 Skill 调优方案

## 核心问题诊断

当前 skill 把四个层级混在一起了：
1. 规则判断（危险品要不要单独）
2. 经验建议（易碎品注意保护）
3. 物理装箱（到底摆不摆得下）
4. 物流计费（按实际重还是体积重）

## 调优目标（三层）

1. 先让 skill 不胡说（区分建议 vs 精确结果）
2. 再让 skill 真的会算（规则分组 + 物理校验）
3. 最后让 skill 输出能解释、能验收（审计字段 + 数据来源标注）

## 四阶段优先级

### 第一阶段：诚实模式（最高优先）
- 区分 rule_based_estimation / physical_packing 两种模式
- 缺主数据时禁止输出精确 fill_rate / 计费重 / 验证通过
- 增加数据缺失说明和可信度评分

### 第二阶段：规则分组优化
- 硬规则：dangerous_goods 单独组、liquid 不与 dangerous_goods 同组、oversize 单独组
- 软规则：fragile 尽量不与重货混组、liquid 尽量立放、compressible 不压 fragile
- 先分组后装箱，再尝试组间合并优化

### 第三阶段：物理装箱校验
- 单件尺寸校验（可旋转后能否放入）
- 箱重校验（含包材重量）
- 缓冲空间系数（fragile ×1.15, liquid ×1.10, 普通 ×1.00）
- 简易摆放逻辑（不只看体积，还看长宽高布局）

### 第四阶段：结果可解释
- 每个包裹输出：为什么独立成包、为什么选这个箱、为什么没选更小箱
- 数据来源标注：explicit_input / catalog_lookup / inferred / unknown
- 三类结论分开输出：规则校验 / 物理装箱 / 计费结果
