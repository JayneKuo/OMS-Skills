# 时效引擎

ETA（Estimated Time of Arrival，预估送达时间）是分仓决策的核心输入之一。本引擎禁止仅依赖承运商 API 返回值作为 ETA，必须实现混合校准模型。

## 7.5.1 ETA 公式

ETA 由以下 8 个组件累加得出：

```
ETA_total = T_queue + T_cutoff_wait + T_process + T_handover + T_transit + T_last_mile + T_weather + T_risk_buffer
```

各组件说明：

| 编号 | 组件 | 中文名 | 说明 |
|------|------|--------|------|
| 1 | T_queue | 排队等待时间 | 订单进入仓库作业队列后，等待开始拣货的时间，与仓当前积压量相关 |
| 2 | T_cutoff_wait | 截单等待时间 | 如果订单在截单时间之后到达，需等待到下一个作业窗口的时间 |
| 3 | T_process | 仓内处理时间 | 拣货、复核、打包、称重、贴面单的总时间 |
| 4 | T_handover | 交接时间 | 从仓库打包完成到承运商实际揽收的等待时间 |
| 5 | T_transit | 干线运输时间 | 承运商从揽收到目的地转运中心的运输时间 |
| 6 | T_last_mile | 末端配送时间 | 从目的地转运中心到客户签收的时间 |
| 7 | T_weather | 天气影响时间 | 因极端天气（暴雨、暴雪、台风）导致的额外延迟 |
| 8 | T_risk_buffer | 风险缓冲时间 | 基于历史数据的风险缓冲，覆盖不可预见的延迟 |

## 7.5.2 ETA 必须考虑的因素

以下 8 个因素必须纳入 ETA 计算，缺少任何一个都会导致 ETA 过度乐观：

| 编号 | 因素 | 说明 |
|------|------|------|
| 1 | 截单时间（Cut-off Time） | 超过截单时间的订单，最早发货日顺延至下一工作日 |
| 2 | 仓作业班次 | 仓库的作业时间窗口（如 8:00-22:00），非作业时间不计入处理时间 |
| 3 | 仓当前积压量 | 仓库当前待处理订单数量，影响排队时间 |
| 4 | 承运商揽收班次 | 承运商每日揽收次数和时间点，错过揽收需等待下一班次 |
| 5 | 承运商历史准点率 | 该承运商在该线路上的历史准点率，准点率低时需增加缓冲 |
| 6 | 节假日与休息日 | 法定节假日、周末对仓库作业和物流运输的影响 |
| 7 | 天气预警 | 目的地或途经地区的天气预警信息 |
| 8 | 大促期间拥堵 | 大促期间（如双 11、618）物流网络整体拥堵导致的额外延迟 |

## 7.5.3 风险化口径

ETA 的风险化处理分为 3 个层级：

| 层级 | 口径 | 说明 |
|------|------|------|
| 乐观 ETA | P50 | 50% 概率能在此时间内送达，用于前端展示"最快送达" |
| 标准 ETA | P75 | 75% 概率能在此时间内送达，用于分仓决策的默认口径 |
| 保守 ETA | P90 | 90% 概率能在此时间内送达，用于 SLA 承诺和客诉判定 |

分仓决策默认使用标准 ETA（P75）进行计算，但在以下场景切换口径：
- 客户为 VIP 等级：使用保守 ETA（P90）确保履约承诺
- 大促期间：使用保守 ETA（P90）预留缓冲
- 前端展示：使用乐观 ETA（P50）提升用户体验

## 7.5.4 ETA 分桶口径

ETA 的历史数据按以下 6 个维度进行分桶统计：

| 维度 | 分桶粒度 | 说明 |
|------|----------|------|
| 发货仓 | 仓库 ID | 每个仓库独立统计 |
| 承运商 | 承运商 ID | 每个承运商独立统计 |
| 目的地 | 省/市/区 | 按行政区划三级分桶 |
| 时间段 | 工作日/周末/节假日 | 区分不同时间段的时效差异 |
| 商品类型 | 标准件/大件/生鲜 | 不同商品类型的处理时效不同 |
| 季节 | 春/夏/秋/冬 | 季节性因素对物流时效的影响 |

4 级回退层级（当某个分桶样本不足时）：

| 回退级别 | 分桶口径 | 说明 |
|----------|----------|------|
| Level 1 | 仓 × 承运商 × 区 × 时间段 × 商品类型 × 季节 | 最细粒度，优先使用 |
| Level 2 | 仓 × 承运商 × 市 × 时间段 | 放宽目的地到市级，忽略商品类型和季节 |
| Level 3 | 仓 × 承运商 × 省 | 放宽目的地到省级，忽略时间段 |
| Level 4 | 承运商 × 省 | 最粗粒度，跨仓汇总 |

## 7.5.5 样本不足回退逻辑

当历史数据样本不足时，按以下 4 步回退：

| 步骤 | 条件 | 动作 |
|------|------|------|
| 1 | 当前分桶样本数 ≥ min_sample（默认 30） | 使用当前分桶的统计值 |
| 2 | 当前分桶样本数 < min_sample | 向上回退一级分桶（参见 7.5.4 回退层级） |
| 3 | 所有分桶样本数都 < min_sample | 使用承运商 API 返回值 × 校准系数（默认 1.2） |
| 4 | 承运商 API 不可用 | 使用系统默认 ETA（按距离分段：同城 1 天，省内 2 天，跨省 3 天，偏远 5 天） |

## 7.5.6 风险修正

ETA 风险修正用于在基础 ETA 上叠加风险因子：

```
ETA_adjusted = ETA_base × (1 + risk_factor)
```

风险因子的计算：

```
risk_factor = max(f_weather, f_congestion, f_carrier_risk)
```

| 风险因子 | 触发条件 | 默认值 |
|----------|----------|--------|
| f_weather | 目的地或途经地有天气预警 | 0.3（暴雨）、0.5（暴雪/台风） |
| f_congestion | 大促期间或物流网络拥堵 | 0.2（普通大促）、0.5（双 11 高峰） |
| f_carrier_risk | 承运商近 7 天准点率 < 80% | (1 − 准点率) × 0.5 |

## 7.5.7 OnTimeProbability 计算口径

OnTimeProbability（准时送达概率）用于评估某个方案在 SLA 时间内送达的概率：

```
OnTimeProbability = P(ETA_actual ≤ SLA_deadline)
```

基于历史数据的经验分布计算：

```
OnTimeProbability = count(历史订单中 ETA_actual ≤ SLA_deadline) / count(同分桶历史订单总数)
```

当 OnTimeProbability < on_time_threshold（默认 0.85）时，该方案标记为"时效风险"，在综合评分中扣分。

## 7.5.8 典型案例

### 案例：跨省订单 ETA 计算

**场景**：上海仓发货，目的地成都，承运商为顺丰，工作日下单，当前时间 14:00，截单时间 16:00。

**计算过程**：

| 组件 | 取值 | 来源 |
|------|------|------|
| T_queue | 0.5h | 仓当前积压 200 单，历史处理速度 400 单/h |
| T_cutoff_wait | 0h | 当前时间 14:00 < 截单时间 16:00，无需等待 |
| T_process | 2h | 该仓标准件平均处理时间 |
| T_handover | 1h | 顺丰下一班揽收时间 17:00，预计 16:30 打包完成 |
| T_transit | 36h | 历史分桶数据：上海仓 × 顺丰 × 成都市 × 工作日，P75 = 36h |
| T_last_mile | 4h | 历史分桶数据：成都市区末端配送 P75 = 4h |
| T_weather | 0h | 无天气预警 |
| T_risk_buffer | 2h | 顺丰该线路准点率 92%，风险缓冲 = 36 × (1-0.92) × 0.5 ≈ 1.4h，取整 2h |

**ETA_total** = 0.5 + 0 + 2 + 1 + 36 + 4 + 0 + 2 = **45.5 小时 ≈ 1.9 天**

OnTimeProbability（SLA 3 天）= 0.94，满足阈值 0.85，方案可行。
