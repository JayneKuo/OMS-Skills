> 本文档为装箱计算引擎的完整需求规格说明，面向产品、研发、测试和算法实现。  
> 目标是定义一套**可执行、可校验、可解释、可回退**的装箱引擎标准，支持规则建议、严格装箱、计费计算与异常回退。

---

# 1. 文档目标

装箱引擎用于根据订单商品、SKU 主数据、箱规、承运商约束和商家包装偏好，自动生成装箱方案，并输出每个包裹的箱型、商品明细、计费重量、包裹数量及异常信息。

本引擎需支持以下能力：

1. 按业务硬规则完成商品分组与隔离
2. 基于箱型与物理属性完成装箱计算
3. 基于承运商规则完成计费重量计算
4. 在装箱失败时提供标准回退机制
5. 输出可审计、可解释的装箱结果
6. 区分“规则建议型结果”与“严格物理装箱结果”

---

# 2. 装箱引擎输出级别

装箱引擎不是所有场景都能输出“严格验证通过”的结果，因此必须先判断输入完整性，并决定输出级别。

## 2.1 输入完整性分级

| 级别 | 条件 | 输出级别 |
|------|------|----------|
| L1 | 仅有 SKU 名称、数量、自然语言属性，缺少尺寸/重量/箱规等关键字段 | 只能输出规则建议型结果 |
| L2 | 有部分 SKU 主数据和箱规，但部分承运商限制或包装参数缺失 | 可输出近似装箱结果，必须标记为估算 |
| L3 | SKU 主数据、箱规、承运商限制、包装偏好完整 | 可输出严格装箱结果 |

## 2.2 输出状态定义

| 状态 | 说明 |
|------|------|
| SUCCESS | 严格装箱成功，规则和物理校验均通过 |
| ESTIMATED | 基于部分缺失数据、经验参数或虚拟箱型计算出的估算方案 |
| CARRIER_SWITCHED | 原承运商不满足约束，切换承运商后得到可行方案 |
| MANUAL_REVIEW_REQUIRED | 系统无法自动确定最终方案，需要人工介入 |
| FAILED | 无可行装箱方案 |

## 2.3 校验结果分层

装箱引擎必须分别输出以下两类校验结果：

1. **规则校验结果**
   - 温区规则
   - 危险品规则
   - 禁混规则
   - 易碎品规则
   - 液体规则

2. **物理校验结果**
   - 商品是否可装入箱内
   - 是否超重
   - 是否超尺寸
   - 是否满足摆放方向要求
   - 是否满足缓冲空间要求

> 规则校验通过，不代表物理装箱成功。  
> 物理装箱成功，也必须同时满足业务规则，才可视为最终成功。

# 3. 装箱输入

装箱计算需要以下输入数据。

## 3.1 核心输入项

| 编号 | 输入项 | 说明 |
|------|--------|------|
| 1 | SKU 列表及数量 | 订单中每个 SKU 的数量 |
| 2 | SKU 物理属性 | 每个 SKU 的长、宽、高、实际重量 |
| 3 | SKU 品类属性 | 温区、是否危险品、是否易碎、是否液体 |
| 4 | SKU 混装限制 | 是否允许与其他 SKU 混装、禁混品类列表 |
| 5 | 可用箱型列表 | 该仓可用的标准箱型及内部尺寸、外部尺寸、最大承重 |
| 6 | 填充率要求 | 最低填充率阈值，默认 60% |
| 7 | 承运商限制 | 承运商对单包裹重量、尺寸、体积、液体总量等限制 |
| 8 | 商家包装偏好 | 是否需要礼盒、防震、防漏、温控、品牌包装等偏好 |

## 3.2 SKU 物理属性扩展字段

为保证装箱结果可执行，SKU 物理属性除长宽高和重量外，建议增加以下字段：

| 字段 | 说明 |
|------|------|
| rotate_allowed | 是否允许旋转摆放 |
| upright_required | 是否必须保持竖直 |
| stackable | 是否可叠放 |
| compressible | 是否可压缩 |
| max_stack_layers | 最大叠放层数 |
| fragile_level | 易碎等级 |
| liquid_volume_ml | 液体容量，单位 ml |

## 3.3 SKU 品类属性标准

| 字段 | 值示例 |
|------|--------|
| temperature_zone | ambient / chilled / frozen |
| dangerous_goods | true / false |
| fragile | true / false |
| liquid | true / false |

## 3.4 箱型数据结构

每种箱型应至少配置以下信息：

| 字段 | 说明 |
|------|------|
| box_type_code | 箱型编号 |
| inner_length / inner_width / inner_height | 内部尺寸，用于装箱校验 |
| outer_length / outer_width / outer_height | 外部尺寸，用于承运商尺寸校验 |
| max_weight_kg | 最大承重 |
| tare_weight_kg | 箱体自重 |
| cushioning_supported | 是否支持防震填充 |
| leakproof_supported | 是否支持防漏包装 |
| temperature_zone_supported | 支持的温区 |
| available_qty | 仓内可用数量 |

## 3.5 承运商限制

| 字段 | 说明 |
|------|------|
| max_package_weight_kg | 单包裹最大重量 |
| max_package_length_cm | 最大边长 |
| max_package_girth_cm | 最大围长 |
| max_package_volume_cm3 | 最大体积 |
| volumetric_divisor | 体积重量系数 |
| max_liquid_volume_ml | 单包液体总量上限 |
| special_box_types_supported | 支持的大件/冷链/特殊箱型列表 |

## 3.6 包材与附加包装参数

包装材料不得视为固定常量，必须可配置。

| 字段 | 说明 |
|------|------|
| box_tare_weight_kg | 箱体默认自重 |
| cushion_weight_kg | 防震填充重量 |
| cushion_volume_cm3 | 防震填充体积占用 |
| leakproof_weight_kg | 防漏包装重量 |
| leakproof_volume_cm3 | 防漏包装体积占用 |
| gift_wrap_weight_kg | 礼盒/礼品包装重量 |
| gift_wrap_volume_cm3 | 礼盒/礼品包装体积占用 |

# 4. 装箱硬规则

以下规则为硬规则，不可违反。违反则当前装箱方案无效。

| 编号 | 规则 | 说明 |
|------|------|------|
| 1 | 温区不混装 | 常温、冷藏、冷冻商品不可装入同一包裹 |
| 2 | 危险品隔离 | 危险品必须单独包装，不可与普通商品混装 |
| 3 | 单包不超重 | 单包裹实际重量不得超过箱型最大承重和承运商最大重量限制 |
| 4 | 单包不超尺寸 | 单包裹外部尺寸不得超过承运商尺寸限制 |
| 5 | 禁混品类隔离 | 标记为禁混的品类不可与其他品类同包 |
| 6 | 易碎品保护 | 易碎品必须使用支持防震填充的箱型，且不可与重物混装 |
| 7 | 液体防漏 | 液体商品必须使用防漏包装，且单包液体总量不得超过承运商限制 |

## 4.1 重物定义

为支持“易碎品不可与重物混装”规则，需定义重物判定标准。

默认规则如下：

1. 单件重量 > 3kg 的商品，视为重物  
2. 若同包内某商品单件重量 > 易碎品单件重量的 2 倍，也可视为重物

该规则需支持仓级或商家级配置覆盖。

## 4.2 液体总量定义

液体总量统一按 `ml` 计算。  
若 SKU 未提供容量但提供重量，则需按主数据换算规则或商家配置的换算系数折算。

# 5. 装箱软规则与优化规则

软规则不直接导致方案失败，但会影响优先级、箱型选择和方案评分。

| 编号 | 规则 | 说明 |
|------|------|------|
| 1 | 最小适配优先 | 在满足规则和装入条件下优先选最小可用箱型 |
| 2 | 填充率优先 | 优先选择填充率在 60%~90% 的方案 |
| 3 | 仓内库存优先 | 优先使用仓内库存充足的箱型 |
| 4 | 承运商适配优先 | 优先选择能被当前承运商直接接受的箱型 |
| 5 | 拆包数最少优先 | 在计费重量接近时，优先包裹数更少的方案 |
| 6 | 防护一致性优先 | 同一包裹内商品的防震、防漏、温控需求尽量一致 |

## 5.1 填充率规则说明

填充率定义如下：

```text
填充率 = 商品总体积（含必要保护空间） / 箱型内部有效容积
````

### 填充率说明

1. 默认阈值为 60%
2. 低于阈值时，应尝试更小箱型
3. 对以下场景允许低填充率豁免，仅标记“低填充率”提示，不视为失败：

   * 危险品单独包装
   * 易碎品需要大量缓冲
   * 液体需要防漏与立放
   * 超长件/特殊件只能使用特殊箱型
   * 温控商品需要保温层

# 6. 装箱执行流程

装箱引擎执行顺序如下。

| 步骤 | 动作        | 说明                          |
| -- | --------- | --------------------------- |
| 0  | 输入完整性检查   | 判断输入级别 L1/L2/L3，并决定输出模式     |
| 1  | SKU 属性标准化 | 标准化尺寸、重量、温区、液体容量、禁混规则       |
| 2  | 品类分组      | 按温区、危险品、禁混规则拆成若干互斥组         |
| 3  | 组内排序      | 每组按体积从大到小排序，采用 FFD 思路生成候选顺序 |
| 4  | 箱型尝试      | 对每组从最小可用箱型开始尝试装箱            |
| 5  | 多包拆分      | 单个箱型装不下时进行拆包                |
| 6  | 填充率与优化    | 检查是否可换更小箱或调整分组              |
| 7  | 承运商校验     | 校验外尺寸、重量、液体总量、体积限制          |
| 8  | 回退与输出     | 无法自动完成时进入回退逻辑并输出方案状态        |

# 7. 装箱算法要求

## 7.1 分组策略

引擎必须先分组，再装箱。
以下因素优先用于分组：

1. 温区
2. 危险品
3. 禁混品类
4. 液体
5. 易碎品
6. 商家指定不可混装标记

分组后的每个组互相独立评估，必要时可在不违反规则的前提下尝试组间合并优化。

## 7.2 候选装箱排序

每个分组内部，默认按商品体积从大到小排序，采用 First Fit Decreasing 生成候选装箱顺序。

> 注意：FFD 仅用于候选顺序生成，不等价于几何装箱成功。
> 所有候选方案都必须经过物理可装入校验，方可认定为成功。

## 7.3 物理可装入校验

候选方案至少需通过以下物理校验：

1. 单件商品可装入箱内

   * 支持旋转时，需尝试合法旋转方向
   * 必须立放商品不得横放
2. 总重量不超过箱型和承运商限制
3. 商品总体积加上必要保护空间后，不超过箱内有效容积
4. 不可叠放商品不得被其他商品压放
5. 易碎品需要预留防震空间
6. 液体商品需要预留防漏和立放空间
7. 需要礼盒/品牌包装时，需加上额外体积与重量

## 7.4 保护空间计算建议

为提升装箱安全性，系统可对不同品类加保护系数：

| 类型     | 默认保护系数 |
| ------ | ------ |
| 普通品    | 1.00   |
| 易碎品    | 1.15   |
| 液体     | 1.10   |
| 礼盒包装   | 1.08   |
| 温控保温包装 | 1.20   |

> 实际保护系数可按商家、仓库、品类配置覆盖。

# 8. 计费规则

## 8.1 实际重量

```text
实际重量 = Σ(SKU_weight × SKU_qty) + 箱体重量 + 附加包装重量
```

其中附加包装重量可包括：

* 防震填充
* 防漏包装
* 礼盒包装
* 保温层

## 8.2 体积重量

```text
体积重量 = (包裹外部长 × 外部宽 × 外部高) / 体积重量系数
```

默认体积重量系数：

| 运输方式 | 系数   |
| ---- | ---- |
| 快递   | 6000 |
| 零担   | 5000 |
| 空运   | 6000 |

## 8.3 计费重量

```text
计费重量 = max(实际重量, 体积重量)
计费重量取整 = ceil(计费重量 × 10) / 10
```

# 9. 标准箱型选择规则

| 编号 | 规则     | 说明                       |
| -- | ------ | ------------------------ |
| 1  | 最小适配原则 | 在满足装入和规则约束前提下，选择最小可用箱型   |
| 2  | 填充率优先  | 优先选择填充率在 60%~90% 区间内的箱型  |
| 3  | 仓内可用优先 | 优先使用该仓库存充足的箱型            |
| 4  | 承运商适配  | 箱型外尺寸必须满足承运商要求           |
| 5  | 防护能力适配 | 易碎品需选支持防震的箱型，液体需选支持防漏的箱型 |
| 6  | 温区适配   | 冷链商品必须选支持对应温区的箱型         |

# 10. 装箱回退机制

当标准装箱流程失败时，按以下级别依次回退：

| 级别 | 触发条件     | 回退动作                   | 输出状态                               |
| -- | -------- | ---------------------- | ---------------------------------- |
| F1 | 标准箱型装不下  | 尝试仓内非标箱型               | SUCCESS 或 ESTIMATED                |
| F2 | 所有箱型都装不下 | 标记“需人工包装”，使用虚拟箱型估算计费重量 | ESTIMATED / MANUAL_REVIEW_REQUIRED |
| F3 | 承运商尺寸超限  | 切换到支持大件的承运商重新计算        | CARRIER_SWITCHED                   |
| F4 | 装箱规则冲突   | 记录冲突详情并挂起，等待人工介入       | MANUAL_REVIEW_REQUIRED             |

# 12. 失败与异常输出要求

当引擎无法输出严格成功方案时，必须明确说明原因，不得只返回“失败”。

## 12.1 常见失败原因

* 缺少 SKU 尺寸
* 缺少 SKU 重量
* 无可用箱型
* 超过承运商尺寸限制
* 温区冲突
* 危险品与禁混品冲突
* 液体总量超过承运商限制
* 需要人工确认包装方式

## 12.2 异常输出要求

1. 必须输出失败原因列表
2. 必须输出已尝试的回退级别
3. 若采用虚拟箱型，必须标记为 `ESTIMATED`
4. 若需人工处理，必须标记为 `MANUAL_REVIEW_REQUIRED`

# 13. 验收标准

## 13.1 功能验收

1. 输入完整时，系统可输出可执行装箱方案
2. 输入不完整时，系统不得虚构精确结果
3. 危险品、温区、禁混规则可正确触发拆包
4. 超重场景可正确拆包
5. 超尺寸场景可正确换箱或切承运商
6. 回退机制触发时，状态正确

## 13.2 数据验收

1. 实际重量计算正确
2. 体积重量计算正确
3. 计费重量取整正确
4. 填充率计算口径一致
5. 包材重量和体积来源可追溯

## 13.3 结果可信度验收

1. 规则校验与物理校验必须分别输出
2. 每个包裹必须能解释“为什么选这个箱”
3. 每个包裹必须能解释“为什么合包/拆包”
4. 相同输入和配置下，多次计算结果应保持稳定

## 案例 5：生鲜混合订单装箱

场景：订单包含 2 件冷冻食品（共 3kg）、1 件冷藏饮料（1.5kg）、3 件常温零食（共 0.8kg）。

装箱过程：

1. 品类分组：冷冻组、冷藏组、常温组
2. 冷冻组：选冷冻 M 号箱（含保温层），计费重量 = max(3.0 + 0.5, 4.2) = 4.2kg
3. 冷藏组：选冷藏 S 号箱，计费重量 = max(1.5 + 0.3, 1.8) = 1.8kg
4. 常温组：选常温 S 号箱，计费重量 = max(0.8 + 0.2, 1.5) = 1.5kg

输出：

* 3 个包裹
* 总计费重量 7.5kg

## 案例 6：超重商品拆包

场景：订单包含 8 件同一 SKU，每件 2.5kg，承运商单包最大重量 15kg。

装箱过程：

1. 总重量 = 8 × 2.5 = 20kg
2. 超过单包限制，拆为两包
3. 包裹 1 装 5 件，实际重量 = 12.5 + 0.5 = 13.0kg
4. 包裹 2 装 3 件，实际重量 = 7.5 + 0.3 = 7.8kg
5. 箱型选择：包裹 1 用 M 号箱，包裹 2 用 S 号箱
6. 计费重量：

   * 包裹 1 = max(13.0, 10.5) = 13.0kg
   * 包裹 2 = max(7.8, 6.3) = 7.8kg

输出：

* 2 个包裹
* 总计费重量 20.8kg
